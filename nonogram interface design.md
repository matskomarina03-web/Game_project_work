Практична робота №22
Індивідуальний проєкт: проєктування інтерфейсу (екрани, навігація, стани UI)
1. Екрани застосунку (MVP)
1.1 Головне меню — навігаційний центр застосунку
        Задача користувача: вибрати рівень або режим гри
        Дані: прогрес, монети, доступні рівні
        Дії:
        вибір раунду Nonogram
        запуск гри
        відкриття MiniGames
        відкриття Maze Adventure
        перегляд прогресу
1.2 Екран гри Nonogram (gamePage)
        Задача: розв’язування логічної головоломки
        Дані: поле гри, підказки, таймер, життя, монети
        Дії:
        заповнення клітинок
        позначення клітинок (X)
        перевірка рішення (Check)
        використання підказки
        скидання поля
1.3 Екран результату (resultPage)
        Задача: показ результату проходження рівня
        Дані: час, помилки, зірки, монети
        Дії:
        перейти на наступний рівень
        повторити рівень
        повернутися в меню
1.4 MiniGames (MiniGamesWidget)
        Задача: вибір додаткових ігор
        Дані: прогрес бонусів
        Дії:
        запуск Maze Adventure
        перегляд прогресу мініігор
        повернення в меню
1.5 Maze Adventure (MazeAdventureWidget)
        Задача: проходження лабіринтів
        Дані: карта рівня, прогрес рівнів
        Дії:
        вибір рівня лабіринту
        переміщення персонажа
        завершення рівня
        повернення назад
2. Структура екранів (UI каркас)
2.1 Main Menu

        Блоки UI:

        Header (назва гри)
        Progress panel (рівні, монети)
        Grid кнопок рівнів
        Кнопки режимів (MiniGames, Maze)

        Центральна область:

        список рівнів (roundButtons)
2.2 Game Screen (Nonogram)

        Блоки UI:

        Top bar (життя, таймер, монети)
        Ігрове поле (boardLayout)
        Панель підказок (row/column hints)
        Панель дій (Hint / Check / Reset)

        Центр:

        ігрове поле Nonogram
2.3 Result Screen

        Блоки UI:

        Заголовок результату
        Статистика
        Оцінка (зірки)
        Кнопки навігації

        Центр:

        результат проходження рівня
2.4 MiniGames

        Блоки UI:

        список мініігор
        кнопки запуску
        прогрес-бар
2.5 Maze Adventure

        Блоки UI:

        список рівнів
        прев’ю лабіринтів
        кнопка “Play”
        ігрове поле (game view)

        Центр:

        MazeGameView
3. Стани екранів
3.1 Main Menu
        звичайний стан: показ рівнів
        заблокований рівень: locked
        вибраний рівень: highlighted
3.2 Game Screen
        порожній стан: новий рівень
        ігровий стан: користувач заповнює поле
        перевірка: підсвітка помилок
        перемога: відкриття картинки
        поразка: 0 lives
3.3 Result Screen
        перемога
        поразка
3.4 Maze
        вибір рівня
        ігровий режим
        завершено рівень
4. Навігаційна схема
        Основні переходи:
        Main Menu → Game Screen (Start Round)
        Game Screen → Result Screen (Check / Finish)
        Result Screen → Game Screen (Next / Repeat)
        Result Screen → Main Menu (Menu)
        Main Menu → MiniGames
        MiniGames → Maze Adventure
        Maze Adventure → Main Menu
        Тригери переходів:
        кнопки (clicked)
        завершення рівня (levelCompleted)
        stack widget (setCurrentWidget)
        Стартовий екран:
        Main Menu (menuPage)
        Повернення назад:
        кнопка “Menu”
        backRequested сигнал
        stackedWidget navigation

7. Узгоджені команди UI
        Основні команди:
        Start
        Check
        Reset
        Hint
        Next
        Repeat
        Menu
        Back
        Play
        Використання:
        Menu: доступна всюди
        Check: тільки в грі
        Hint: тільки під час гри
        Next: тільки після перемоги
        Play: тільки для unlocked рівнів
        Стан доступності:
        disabled якщо:
        рівень locked
        гра завершена
        немає вибраного рівня
8. Висновок

        Було спроєктовано інтерфейс застосунку з чітким розділенням на екрани, стани та навігацію. Основний акцент зроблено на підтримці MVP-сценарію проходження Nonogram, а додаткові модулі (MiniGames та Maze Adventure) інтегровані як розширення функціоналу. Навігація реалізована через стек екранів, що забезпечує просту та логічну структуру переходів.
