Практична робота №21
Індивідуальний проєкт: Nonogram Game + MiniGames + Maze Adventure
1. Тема проєкту (одним реченням)

        Застосунок дозволяє користувачу розв’язувати логічні головоломки типу Nonogram, проходити рівні з поступовим ускладненням та відкривати додаткові міні-ігри, включно з лабіринтом.

2. Постановка задачі

        Цільовий користувач — учень або студент, який хоче розвивати логічне мислення через інтерактивні ігри. У сучасних умовах часто бракує простих офлайн-застосунків, які поєднують навчання та ігрову мотивацію, тому виникає потреба в одному комплексному рішенні.

        Проблема полягає в тому, що користувачі зазвичай використовують окремі додатки для різних типів головоломок, які не мають єдиної прогресії складності та системи мотивації.

        Застосунок вирішує цю проблему, надаючи єдину систему рівнів, монетизацію прогресу (coins), підказки, систему життів та додаткові міні-ігри (Maze Adventure).

        Обмеження: застосунок працює офлайн, дані зберігаються локально через QSettings, платформа — desktop (Qt Widgets, C++).

3. Сутності та дані
3.1 Round (основна сутність рівня Nonogram)
        Обов’язкові поля:
        number (int) — номер рівня
        size (int) — розмір поля
        solution (QVector<QVector<int>>) — правильне рішення
        difficulty (QString) — складність
        unlocked (bool) — доступність рівня
        Опційні поля:
        bestSeconds (int)
        bestStars (int)
        coinsEarned (int)
        Ідентифікація: number
3.2 CellState (стан клітинки)
        Unknown
        Filled
        Marked
        Ідентифікація: координати (row, column)
3.3 ProgressData (прогрес міні-ігор)
        Обов’язкові:
        completedNonogramLevels (int)
        unlockedMazeLevel (int)
        Опційні:
        completedMazeLevels (list)
        Ідентифікація: внутрішній стан сервісу
3.4 PlayerProgress (глобальний прогрес)
        coins (int)
        dailyMissionDone (bool)
        lives (int)
4. Екрани застосунку
4.1 Головне меню (menuPage)
        Призначення: вибір режиму та рівня
        UI:
        кнопки рівнів
        кнопки “Мінігра”, “Лабіринт”
        прогрес проходження
        Дані:
        кількість пройдених рівнів
        монети
4.2 Екран гри Nonogram (gamePage)
        Призначення: розв’язування головоломки
        UI:

        підказки рядків/стовпців
        кнопки: Hint, Check, Reset

        board стан
        timer
        lives
4.3 Результат рівня (resultPage)
        Призначення: показ результату
        UI:
        зірки
        статистика (час, помилки, монети)
        кнопки Next / Repeat
        Дані:
        результат проходження
4.4 Maze Adventure

        UI:
        карта рівня

        exit кнопка
        Дані:
        рівні лабіринту
        прогрес
4.5 MiniGames
        Призначення: додаткові ігри
        UI:
        список доступних мініігор
        Дані:
        ProgressData
5. Сценарії користувача
5.1 Основний сценарій (Nonogram)
        Користувач відкриває застосунок
        Обирає рівень
        Запускає гру
        Заповнює поле
        Натискає “Check”
        Отримує результат (зірки + монети)
5.2 Створення проходження рівня
        Вибір рівня
        Завантаження поля (loadRound)
        Взаємодія з клітинками
        Перевірка (checkBoard)
        Завершення (finishRound)
5.3 Редагування стану гри
        Користувач змінює клітинки
        Система оновлює board
        UI оновлює стиль клітинки (applyCellStyle)
5.4 Додатковий сценарій (Maze)
        Перехід у MiniGames
        Вибір Maze Adventure
        Запуск рівня
        Переміщення стрілками
        Досягнення фінішу

7. MVP (Minimum Viable Product)
7.1 Головний MVP сценарій

        Повне проходження одного рівня Nonogram:

        вибір рівня → гра → перевірка → результат
7.2 Must-have функції
        система рівнів Nonogram
        ігрове поле з клітинками
        перевірка рішення
        збереження прогресу
        система монет
        базовий UI меню
7.3 Після MVP
        MiniGames система
        Maze Adventure
        щоденні місії
        підказки
        система зірок
        preview completed pictures
        фільтр рівнів
        анімації UI
        додаткові режими складності
8. Критерії готовності MVP
        є меню та екран гри
        рівень можна повністю пройти
        результат коректно визначається
        прогрес зберігається після перезапуску
        немає критичних помилок під час гри
9. Висновок

        Проєкт реалізує повноцінний ігровий застосунок з логічними головоломками, побудований на принципах розділення архітектури (UI/логіка/дані). MVP забезпечує базовий ігровий цикл, а додаткові модулі (Maze Adventure, MiniGames) розширюють функціональність і створюють систему мотивації користувача через прогрес і нагороди.
